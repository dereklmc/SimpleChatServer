Part 1: Consistent Ordering

The chat server connects n number of client nodes with an additional sequencer node (the sever). The server's role is to garauntee messages are delivered in a consistent ordering to all clients. The algorithm is an infinite loop over four, "high-level" steps:

  1. Accept a new connection, if there is any.
  2. Read messages from clients
  3. Write messages to clients.
  4. Close connections that had an error.
